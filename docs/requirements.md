## 1. 项目概述

### 1.1 背景

`docs/assets/` 目录下存放了 4 份课堂小测的 HTML 文件，来自南京大学问卷星平台的「题目正确率」报告页（SingleFile 存档）。涵盖敏捷软件开发核心知识。需转化为可在线答题的静态网站，通过 GitHub Pages 部署。

### 1.2 目标

将 4 份静态报告转化为**每题即时反馈的交互式答题网站**，通过 GitHub Actions 自动部署到 GitHub Pages。

## 2. 源数据资产

### 2.1 清单

| 科目 | 源文件 | 大小 | 题目数 |
|---|---|---|---|
| Scrum | `2026 Scrum.html` | ~577 KB | 20 |
| XP | `2026 XP.html` | ~589 KB | 20 |
| Agile | `2026 Agile.html` | ~576 KB | 17 |
| Kanban & Lean | `2026 Kanban  Lean.html` | ~570 KB | 17 |
| **合计** | | **~2.3 MB** | **74** |

### 2.2 数据特征

- 来源：问卷星报告页（SingleFile 内联保存）
- 题型：全部为单选题（4 选项 A–D）
- 正确答案：选项文本末尾 `&nbsp;` 标记
- 正确率：每题均含百分比统计（90%–100%）
- 含大量 base64 UI 装饰图（已丢弃）

### 2.3 提取结果

通过 `scripts/extract.py` 提取为结构化 JSON，4 个文件共 51 KB，74 题全部含正确答案与正确率，100% 覆盖。

## 3. 功能需求

### 3.1 首页（已实现）

- 科目列表，每项显示标题与描述
- 点击进入对应答题页

### 3.2 答题交互（已实现）

- 每道题显示题号、正文、4 个选项（单选按钮）
- **选项顺序随机打乱**（Fisher-Yates，seed 确定性）
- **选择后即时反馈**：
  - 正确 → 选项绿色高亮 + "回答正确" toast
  - 错误 → 已选红色高亮 + 正确答案绿色高亮 + "回答错误，正确答案是 X" toast
- 锁定题目（不可修改），1 秒后自动跳转下一未答题
- 已锁定的题目保留反馈状态，可回溯查看

### 3.3 导航（已实现）

- 上一题 / 下一题 按钮
- 侧栏答题卡（数字网格，当前/已锁定双色标记）
- 移动端底部圆点导航

### 3.4 成绩（已实现）

- 全部答完后显示「查看成绩」按钮
- 结果页：总分百分比 + 四档评级（优秀/良好/及格/需努力）
- 逐题复盘：每道题显示你的答案 vs 正确答案，错题红色左侧边框

### 3.5 重新作答（已实现）

- 结果页「重新作答」→ 新建引擎，选项重新乱序

## 4. 非功能需求

| 要求 | 实现 |
|---|---|
| 零后端依赖 | 纯静态 HTML/CSS/JS/JSON |
| 零框架 | ES Modules，无 React/Vue |
| 响应式 | 桌面双栏 / 移动单栏，断点 500px / 680px |
| 无运行时构建 | GitHub Pages 直接部署 `docs/` |
| 可访问性 | 语义化 HTML、`role` 属性、键盘导航 |
| 数据可维护 | JSON 驱动，新增科目只需添加 JSON 文件 |

---

## 5. 技术实现

| 层 | 技术 |
|---|---|
| 结构 | HTML5（单页 SPA，Hash 路由） |
| 样式 | CSS3 自定义属性（Anthropic 风格配色） |
| 逻辑 | Vanilla JS（ES Modules，3 个模块） |
| 数据 | JSON（4 个文件，`docs/data/`） |
| 提取 | Python 脚本（`scripts/extract.py`，一次性） |

## 6. 部署方案

```
GitHub Repository
  └─ .github/workflows/deploy.yml
       push main → checkout → upload artifact (docs/) → deploy to Pages
```

**URL**: `https://<user>.github.io/<repo>/`
